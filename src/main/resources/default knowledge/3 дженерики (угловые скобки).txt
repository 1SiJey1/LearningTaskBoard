Тк я уже довольно давно прошу тебя забить на них, то сейчас разберемся по подробнее, что такое эти угловые скобки и
зачем они нужны.

Мне чет в голову пока не приходят идеи, что бы такое придумать, чтобы это в нашу идею с таск боардом хорошо влетело...
О, придумал!

1. Надо сделать новый package (пакет), с названием
com.evgeniy.task.reward
https://www.youtube.com/watch?v=7VZRmUjuS1g - видос про джава пакеты

2. Создать там интерфейс Reward<T>, где T - тип вознаграждения
    В нем должен быть метод
        T getActualReward();
3. Реализовать класс DefaultReward<T> implements Reward<T>
    В нем должно быть поле
        private final T reward;

4. Интерфейсу Task добавить новый метод
    Reward<T> getReward();

5. Изменить интерфейс Task и класс DefaultTask соответствующим образом, чтобы их названия не горели желтым,
    т.е. не было сообщения Raw use of parameterized class.

Возможно к этому моменту ты слегка запутаешься, поэтому вот видос:
https://www.youtube.com/watch?v=5gU2wrQnbJs&ab_channel=Followthewhiterabbit - видос про дженерики (обобщения)

6. Reward не надо добавлять в методы equals/compareTo. Но надо добавить в toString, также надо чтобы DefaultReward
    тоже бы реализовывал метод toString

7. Изменить метод TaskCreationService#createTest, чтобы у него было такое объявление:
public <T> Task<T> createTask(String name, String description, Reward<T> reward)

прикол в том, что можно угловые скобки указывать не только на уровне класса (например, Reward<T>), но и на уровне
метода (как в этом случае).

соответственно если где-то что-то будет непонятно - спрашивай,
все написанные юнит тесты после этого задания должны успешно выполняться
если в каком-то тесте тебе поле reward не нужно, то передавай туда null
